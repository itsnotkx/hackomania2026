  <action>
Add the /ws WebSocket endpoint to backend/main.py. The endpoint MUST use asyncio.to_thread() for inference — this is the single highest-impact architectural constraint, not optional.

Add to main.py:

```python
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
import asyncio
import time
from inference import load_model, is_ready, run_inference, MODEL_ID, CHUNK_BYTES

@app.websocket("/ws")
async def websocket_endpoint(ws: WebSocket):
    await ws.accept()
    try:
        while True:
            data = await ws.receive_bytes()
            # CRITICAL: inference runs in thread pool — never on event loop directly
            result = await asyncio.to_thread(run_inference, data)
            await ws.send_json(result)
    except WebSocketDisconnect:
        pass  # Client disconnected cleanly — not an error
```

Behavior requirements:
- Accept binary frames only (receive_bytes, not receive_text)
- MUST use asyncio.to_thread(run_inference, data) — calling run_inference(data) directly in the async handler would freeze all connections for 400-600ms per chunk
- Send JSON response via send_json() (handles serialization automatically)
- Handle WebSocketDisconnect gracefully — client disconnecting is normal, not an error
- No authentication, no session state — each frame is fully independent
- Do NOT add a length check that rejects non-64000-byte frames in v1 — accept any length and let the model handle it (flexibility for test scripts sending silence)

The lifespan handler and /health endpoint from Plan 01 must remain unchanged.
