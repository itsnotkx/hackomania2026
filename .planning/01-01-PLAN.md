Implement `run_inference(audio_bytes: bytes) -> dict`:

- Measure inference wall-clock time in ms using `time.perf_counter()`
- Return `{"label": "AI" if score > DEFAULT_THRESHOLD else "HUMAN", "score": round(score, 4), "ms": round(inference_ms)}`
- Do NOT call this function from async context directly — caller must use asyncio.to_thread()

Use `from __future__ import annotations` and standard library imports only (numpy, torch, transformers, time).

  <action>
Create backend/main.py:

```python
from fastapi import FastAPI
from contextlib import asynccontextmanager
import asyncio
from inference import load_model, is_ready, MODEL_ID

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Run model loading in thread pool — transformers download can take 30s on cold start
    await asyncio.to_thread(load_model)
    yield

app = FastAPI(lifespan=lifespan)

@app.get("/health")
async def health():
    return {
        "status": "ready" if is_ready() else "loading",
        "model": MODEL_ID
    }
```

Use the lifespan context manager pattern (NOT the deprecated @app.on_event("startup") decorator — FastAPI 0.111 deprecates it).

Run model loading via asyncio.to_thread(load_model) to avoid blocking the event loop during the potentially slow model download + warmup.

The /health endpoint MUST reflect model_ready state — not just HTTP 200. A client calling /health before the model finishes loading must receive {"status": "loading"}, not {"status": "ready"}.

Start the server locally to verify: `cd backend && uvicorn main:app --host 0.0.0.0 --port 8000`
